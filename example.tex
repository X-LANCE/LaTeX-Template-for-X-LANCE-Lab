\abstract{
This is a \LaTeX{} document template designed for technical reports, preprints produced by the OpenDFM team at X-LANCE Lab, Shanghai Jiao Tong University. The template is built on top of the standard \texttt{article} class and provides a cohesive set of typographic defaults, including Palatino body text, a unified color theme, and a carefully styled title box with author affiliations and metadata links. It ships with four built-in code-listing styles (Python, JSON, Bash, and plain-text prompt), a \texttt{promptbox} environment tailored for LLM-style dialogues, and full support for AMS math environments, \texttt{booktabs} tables, and \texttt{subcaption} sub-figures. References are managed via \texttt{natbib} with an IEEE-compatible bibliography style. This companion document serves as both a user guide and a living example: every feature described herein---equations, figures, tables, code blocks, colored boxes, and cross-references---is demonstrated in place so that authors can copy and adapt the source directly.
}

\subsection{Python}

\begin{lstlisting}[style=python, caption={A simple Python example}]
import torch
import torch.nn as nn

class TransformerBlock(nn.Module):
    """A single Transformer encoder block."""

    def __init__(self, d_model: int = 512, n_heads: int = 8):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, n_heads)
        self.norm = nn.LayerNorm(d_model)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_model * 4),
            nn.GELU(),
            nn.Linear(d_model * 4, d_model),
        )

    def forward(self, x):
        # Self-attention with residual connection
        attn_out, _ = self.attn(x, x, x)
        x = self.norm(x + attn_out)
        return x + self.ffn(x)

if __name__ == "__main__":
    model = TransformerBlock()
    dummy = torch.randn(10, 32, 512)  # (seq_len, batch, d_model)
    output = model(dummy)
    print(f"Output shape: {output.shape}")
\end{lstlisting}

\section{Template Overview} \label{section:overview}

The \texttt{opendfm} class is a \LaTeX{} document template jointly maintained by X-LANCE Lab and OpenDFM, designed for technical reports, preprints, and academic papers. It is built on the standard \texttt{article} class and ships with the following built-in features:

\begin{itemize}
    \item \textbf{Typography.} Body text is set in Palatino (\texttt{tgpagella} + \texttt{mathpazo}); math symbols inherit the same family for a consistent look; monospace text uses Inconsolata.
    \item \textbf{Color theme.} A curated palette (\texttt{odlblue}, \texttt{odlbg}) is predefined in the class file. Hyperlinks, citations, and URLs are uniformly colored.
    \item \textbf{Code listings.} Four ready-to-use \texttt{lstlisting} styles are provided: \texttt{python}, \texttt{json}, \texttt{bash}, and \texttt{prompt}, plus a \texttt{promptbox} tcolorbox environment for LLM-style prompts.
    \item \textbf{Math support.} Full AMS math environments are available, including inline formulas, display equations, aligned multi-line equations, piecewise functions, and matrices.
    \item \textbf{Floats.} Standard \texttt{figure} and \texttt{table} environments are supported, with sub-figures via \texttt{subcaption} and publication-quality tables via \texttt{booktabs}.
    \item \textbf{Bibliography.} References are managed by \texttt{natbib} + \texttt{IEEEtran2.bst}, producing numeric citations (\eg \cite{vaswani2017attention}) compatible with IEEE style.
\end{itemize}

% ===== Mathematical Equations =====
\section{Mathematical Equations} \label{section:math}

\subsection{Inline Equations}

Inline math is enclosed in single dollar signs. For example, given a sequence of length $n$ and hidden dimension $d$, the time complexity of self-attention is $\mathcal{O}(n^2 d)$, while the feed-forward sublayer costs $\mathcal{O}(n d^2)$. When $n \ll d$ the bottleneck lies in the feed-forward layer; when $n \gg d$ the quadratic term of attention dominates.

\subsection{Display Equations}

Scaled dot-product attention \cite{vaswani2017attention} is defined as:

\begin{equation}
    \operatorname{Attention}(Q, K, V)
    = \operatorname{softmax}\!\left(\frac{QK^{\top}}{\sqrt{d_k}}\right) V
    \label{eq:attention}
\end{equation}

where $Q \in \mathbb{R}^{n \times d_k}$, $K \in \mathbb{R}^{m \times d_k}$, and $V \in \mathbb{R}^{m \times d_v}$ are the query, key, and value matrices respectively, and $d_k$ is the key dimension used for gradient scaling.

\subsection{Multi-line Aligned Equations}

Multi-head attention concatenates $h$ independent attention heads and applies a linear projection:

\begin{align}
    \operatorname{MultiHead}(Q, K, V)
    &= \operatorname{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\, W^O
    \label{eq:multihead} \\
    \text{where}\quad \mathrm{head}_i
    &= \operatorname{Attention}(Q W_i^Q,\; K W_i^K,\; V W_i^V)
    \notag
\end{align}

with learnable projections $W_i^Q \in \mathbb{R}^{d_{\mathrm{model}} \times d_k}$, $W_i^K \in \mathbb{R}^{d_{\mathrm{model}} \times d_k}$, $W_i^V \in \mathbb{R}^{d_{\mathrm{model}} \times d_v}$, and $W^O \in \mathbb{R}^{h d_v \times d_{\mathrm{model}}}$.

\subsection{Piecewise Functions and Matrices}

The ReLU activation can be written as a piecewise function:

\begin{equation}
    \operatorname{ReLU}(x) = \max(0, x) =
    \begin{cases}
        x, & x > 0 \\
        0, & x \leq 0
    \end{cases}
    \label{eq:relu}
\end{equation}

A $3 \times 3$ rotation matrix about the $z$-axis is:

\begin{equation}
    R(\theta) =
    \begin{pmatrix}
        \cos\theta & -\sin\theta & 0 \\
        \sin\theta &  \cos\theta & 0 \\
        0          &  0          & 1
    \end{pmatrix}
    \label{eq:rotation}
\end{equation}

Cross-references are produced with \verb|\cref{}|, \eg \cref{eq:attention,eq:multihead,eq:relu,eq:rotation}.
